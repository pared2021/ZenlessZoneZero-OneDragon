# 通知配置

通知配置由 `NotifyConfig` 维护，运行时由 `operation_notify.py` 根据当前应用读取。

## 配置维度

通知语义拆成两个平行维度：

- 应用通知：控制应用生命周期消息，取值为关闭、仅结束、开始和结束。
- 节点通知：控制运行过程中的节点细节，取值为关闭、仅失败、逐条、合并。

`merge_error_immediate_notify` 只在节点通知为“合并”时生效，用于失败节点额外立即通知。

一条龙运行页中的“通知设置”使用推入式滚动设置界面，不再通过弹窗编辑。配置变更在控件值变化时立即写入。
应用卡片“更多”菜单中的“通知设置”使用 Flyout，只编辑当前应用的应用通知和节点通知。

## 旧配置迁移

main 分支旧版配置使用应用 ID 对应的整数值：

- `0`：关闭
- `1`：仅应用
- `2`：全部（逐条）
- `3`：全部（合并）

启动构造 `NotifyConfig` 时，如果 YAML 中没有 `applications` 新结构，会立即迁移并落盘：

- `0` -> 应用通知关闭，节点通知关闭
- `1` -> 应用通知按旧 `enable_before_notify` 映射，节点通知按旧 `notify_on_error` 映射为仅失败或关闭
- `2` -> 应用通知按旧 `enable_before_notify` 映射，节点通知逐条
- `3` -> 应用通知按旧 `enable_before_notify` 映射，节点通知合并

旧字段保留在文件中作为迁移来源；迁移完成后，运行时只读取
`applications.<app_id>.lifecycle` 和 `applications.<app_id>.detail`。
框架内不再暴露旧版整数通知等级 API。
